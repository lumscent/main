# MyLink Known Problems

Статус: `draft`  
Продукт: `MyLink`

---

## Назначение

Файл хранит известные и предполагаемые UX-проблемы MyLink.

Используется, чтобы:
- учитывать проблемные зоны при проектировании;
- не повторять уже найденные ошибки;
- формировать гипотезы для исследований;
- связывать задачи с пользовательскими проблемами.

---

## Статусы

| Статус | Значение |
|---|---|
| `hypothesis` | Предположение, требует проверки |
| `validated` | Подтверждено исследованием или аналитикой |
| `in-progress` | В работе |
| `resolved` | Решено |

---

## Проблемы и гипотезы

| Проблема | Зона продукта | Роль | Статус | Что проверить |
|---|---|---|---|---|
| Пользователь может не сразу понять ценность MyLink | Главная страница | Кандидат | `hypothesis` | Понятно ли с первого экрана, что это за сервис и зачем им пользоваться |
| Пользователь может не понимать, что произойдёт после отклика | Отклик | Кандидат | `hypothesis` | Объясняется ли, куда отправлен отклик и какой следующий шаг |
| Карточка вакансии может не давать достаточно информации для решения | Список вакансий / карточка | Кандидат | `hypothesis` | Видны ли ключевые условия: должность, город, зарплата, график, работодатель |
| Форма отклика может быть слишком длинной или непонятной | Форма отклика | Кандидат | `hypothesis` | Какие поля обязательны и понятно ли, зачем они нужны |
| Регистрация может быть лишним барьером перед откликом | Регистрация / авторизация | Кандидат | `hypothesis` | Можно ли откликнуться без регистрации или понятно ли объяснена её польза |
| Пустое состояние поиска может не помогать продолжить сценарий | Поиск / фильтры | Кандидат | `hypothesis` | Есть ли подсказки: изменить запрос, сбросить фильтры, посмотреть похожие вакансии |
| Пользователь может не доверять сервису при передаче данных | Форма / отклик | Кандидат | `hypothesis` | Понятно ли, кто получит данные и как они будут использованы |
| Фильтры могут не соответствовать реальной логике выбора работы | Поиск / фильтры | Кандидат | `hypothesis` | Есть ли фильтры по городу, зарплате, графику, опыту, типу занятости |
| Пользователь может не понимать актуальность вакансии | Вакансия | Кандидат | `hypothesis` | Видно ли, активна ли вакансия и когда она обновлялась |
| Ошибка может не объяснять, как продолжить | Ошибки / отправка формы | Кандидат | `hypothesis` | Есть ли понятный текст ошибки и действие для восстановления сценария |

## Как Claude должен использовать файл

Claude должен:

- использовать этот файл как список гипотез и проблемных зон;
- не считать гипотезы подтверждёнными фактами;
- связывать рекомендации с конкретной проблемой;
- предлагать, какие данные нужно собрать для проверки;
- обновлять статус проблемы только при наличии исследования, аналитики или подтверждённого фидбека.
